# gjw 使用教程 —— 从配置到上手

## ⚠️ 最重要的前提

**gjw 只能在 Cursor 的 Agent 模式下工作。**

普通 Chat 模式（Ask/Chat）下，AI 不会调用 MCP 工具，也就不会调用 `check_messages`，浏览器发的消息 AI 永远收不到。

---

## 第一步：确认配置正确

### 1.1 检查 mcp.json

打开你当前 Cursor 项目的 `.cursor/mcp.json`（也就是 `/home/xut/csclip/.cursor/mcp.json`），内容应该是：

```json
{
  "mcpServers": {
    "gjw-mcp": {
      "command": "node",
      "args": ["/home/xut/csclip/gjw/server.js"]
    }
  }
}
```

**注意**：`args` 必须是 `server.js` 的**绝对路径**，不能是相对路径。

### 1.2 检查规则文件

确认 `/home/xut/csclip/.cursor/rules/` 目录下有这两个文件：
- `gjw.mdc`
- `mcp-messenger.mdc`

如果缺少，从 `gjw/.cursor/rules/` 复制过来。

### 1.3 重启 Cursor

修改配置后必须重启：
- `Ctrl+Shift+P` → `Developer: Reload Window`
- 或者完全关闭 Cursor 再打开

---

## 第二步：启动 gjw 服务

在终端运行：

```bash
cd /home/xut/csclip/gjw
node server.js
```

看到输出 `[gjw-mcp] Chat UI ready: http://localhost:3456` 说明启动成功。

**注意**：这个终端要保持运行，关闭终端 = 关闭服务。

---

## 第三步：打开浏览器界面

访问 http://localhost:3456

你应该看到：
- 左侧：会话列表（默认显示"默认会话"）
- 中间：聊天区域
- 下方：输入框
- 右上角：状态 pill（蓝色"AI 处理中"或绿色"等待输入..."）

---

## 第四步：在 Cursor 中发起 Agent 对话（关键！）

### 4.1 切换为 Agent 模式

在 Cursor Chat 面板左上角，确认模式选择为 **Agent**（不是 Ask，不是 Chat）。

### 4.2 发一条任务

例如：
> "帮我写一个 Python 函数，计算斐波那契数列"

### 4.3 观察 AI 的行为

AI 处理完成后，**最后一步应该显示调用了 `check_messages` 工具**。

如果 AI 直接结束了对话（没有工具调用），说明规则没有生效，回到第一步检查配置。

---

## 第五步：在浏览器中发消息

当 AI 调用 `check_messages` 后：

1. 浏览器右上角状态从 **"AI 处理中"（蓝色）** 变成 **"等待输入..."（绿色闪烁）**
2. 在浏览器底部输入框输入后续指令
3. 按 Enter 或点击发送按钮

例如：
> "把函数改成支持缓存的版本"

4. AI 收到消息后自动继续工作
5. 工作完成后再次调用 `check_messages`
6. 状态再次变绿，等待下一条消息
7. 循环往复

---

## 状态对照表

| 浏览器状态 | 颜色 | 含义 | 你能做什么 |
|-----------|------|------|-----------|
| 连接中... | 灰色 | 正在连接 server | 等待 |
| AI 处理中 | 蓝色 | AI 正在工作 | 等待 |
| 等待输入... | 绿色（闪烁） | AI 已调用 check_messages，正在监听 | **发消息！** |
| 未连接 | 灰色 | server 挂了或网络断开 | 检查 server.js 是否运行 |

## 常见问题排查

### Q1: AI 回复后直接结束，没有调用 check_messages

**原因**：你使用的是普通 Chat/Ask 模式，不是 Agent 模式。

**解决**：切换到 Agent 模式再发消息。

### Q2: 浏览器状态一直是"AI 处理中"，不变绿

**原因**：AI 没调用 check_messages，或者 server 没有收到 MCP 连接。

**排查步骤**：
1. 确认是 Agent 模式
2. 确认 server.js 正在运行（`ps aux | grep server.js`）
3. 确认 mcp.json 路径正确（绝对路径）
4. 重启 Cursor（`Developer: Reload Window`）
5. 在 Cursor Settings → MCP 中查看 `gjw-mcp` 是否显示 ✅

### Q3: 发了消息但 AI 没反应

**原因**：AI 调用 check_messages 的时机和浏览器发消息的时机对不上。

**正确时序**：
1. AI 完成回复 → 调用 check_messages → 状态变绿
2. 你发消息 → AI 立即收到
3. AI 处理 → 完成 → 再次调用 check_messages → 状态变绿

**错误时序**：
1. AI 正在处理（蓝色）→ 你发消息 → 消息进队列，AI 当前这轮收不到
2. 等 AI 这轮完成、调用 check_messages 后，才能收到队列里的消息

### Q4: 消息显示重复

**原因**：极罕见的竞态，loadHistory 和 poll 同时返回。

**解决**：刷新浏览器页面。

### Q5: 如何在其他项目中使用 gjw？

把以下文件复制到新项目的 `.cursor/` 目录下：

```
新项目/
└── .cursor/
    ├── mcp.json          # 修改 args 为 gjw/server.js 的绝对路径
    └── rules/
        ├── gjw.mdc
        └── mcp-messenger.mdc
```

然后重启 Cursor 即可。

## 进阶功能

### 多会话

点击左侧「➕ 新建会话」创建新会话，每个会话独立存储消息。

### 消息编辑

将鼠标悬停在自己的消息上 → 点击「编辑」→ 修改内容 → 确认。

编辑后该消息之后的所有内容自动清除，AI 从编辑点重新处理。

### 结构化模板

输入框上方有 7 个模板按钮。选择模板后发送的消息会带有标记，AI 收到后按模板类型处理。

特别地，选择「📓 记到 lab-notes」时，AI 会自动将内容整理后写入项目 `lab-notes/` 目录。

### AI 主动提问

当 AI 需要你做选择时，浏览器会弹出模态框显示问题和选项。支持单选和多选，每道题都可以输入自定义补充文本。
