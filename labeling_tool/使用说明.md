# BALF 细胞标注系统 — 使用说明

## 一、启动系统

### 1. 环境要求
- **Python 环境**: Conda `research_assistant` (Python 3.11)
- **GPU**: CUDA 12.8 + PyTorch 2.10（可选，CPU 也可运行）
- **浏览器**: Chrome / Edge（推荐）

### 2. 启动服务器

**关键：必须从 `D:\VM_share` 目录启动，不能从 `labeling_tool/` 内部启动。**

```bash
cd D:\VM_share
conda activate research_assistant
python -m uvicorn labeling_tool.main:app --host 0.0.0.0 --port 8000 --reload
```

或使用批处理脚本：
```bash
D:\VM_share\labeling_tool\run_server.bat
```

### 3. 访问系统
浏览器打开：**http://localhost:8000**

如看到旧版本界面，请按 `Ctrl+Shift+F5` 强制刷新。

## 三、标注工作流程

### 推荐工作流（4步法）

#### Step 1: CellposeSAM 自动分割
1. 展开右侧 **「① CellposeSAM」** 面板
2. 设置细胞直径（如 `30` 或 `15,60,90` 多尺度）
3. 点击 **「⚡ 运行预览」** → 查看分割结果
4. 满意则点 **「✓ 全部保存到标注」**
5. 批量处理：使用下方 **「一键批量分割」**

#### Step 2: SAM3 精细分割
1. 展开 **「② SAM3 预测」** 面板
2. 先画一个 BBox（按 `R` 进入画框模式）
3. 选择策略：Grounding（框+类名）/ 13点+框 / 纯框
4. 点击 **「⚡ BBox预测」** → SAM3 会对 BBox 内区域做精细分割
5. 接受或放弃预测结果

#### Step 3: 智能分类
1. 展开 **「③ 智能分类」** 面板
2. 选择方法：`Hybrid Adaptive`（推荐）或 `Few-shot`
3. 需要先添加 Support 样本（每类至少1个）：
   - 选中一个已标注的细胞 → 点 **「+ 当前」** 添加为 Support
   - 或使用默认 Support（若已保存）
4. 点 **「一键分类当前图」** 或 **「批量评估」**

#### Step 4: 科研验证
1. 展开 **「④ 科研验证」** 面板
2. 选择金标准标注集
3. 运行 **「分类金标准评估」** 或 **「分割金标准评估」**
4. 查看精度/召回率等指标

## 五、数据管理系统

按 `M` 键或点击底部状态栏高亮的 **「数据管理」** 按钮打开数据管理中心。

### 数据集 Tab
- 查看所有数据集概览（图像数、类别数、标注覆盖率）
- **「打开」**: 将数据集加载到标注页面直接开始标注
- **「分布」**: 展开查看类别分布柱状图
- **「导出」**: 跳转到导出页面
- **「重命名」**: 修改用户添加的数据集名称（自动同步到配置文件）
- **「移除/隐藏」**: 从列表中移除或隐藏数据集（不删除磁盘文件）
- **「添加数据集」**: 通过浏览器选择本地目录，自动检测数据集结构并注册
- 数据集卡片显示标签区分来源（用户添加 / YAML 配置）

### 项目 Tab
- 创建项目，关联多个数据集
- 查看项目仪表盘（总进度、类别分布）
- 项目归档/恢复管理
- 编辑项目名称和描述

### 统计 Tab
- 30天活动总览（标注图像数、创建标注数、AI辅助次数）
- 每日活动柱状图
- 系统概览（总数据集、总图像、标注覆盖率）
- 选择数据集查看类别分布详情（含饼图）

### 评估 Tab
- **快速选择模式**: 从下拉菜单选择数据集、金标准标注集和预测标注集，自动填充路径
- **手动输入模式**: 高级用户可手动输入任意目录路径
- 支持分割评估（Precision/Recall/F1/IoU/Dice）和分类评估（Accuracy/Precision/Recall/F1）
- 显示逐类别详细指标和逐图对比结果

### 导出 Tab
- 支持 5 种格式：**YOLO / COCO / VOC / Mask / CSV**
- YOLO 格式导出为 ZIP 文件自动下载
- 其他格式导出到服务器磁盘
- 选择数据集、标注集、子集（Train/Val）
- 查看导出历史记录

### 审计日志 Tab
- 查看所有操作记录（标注、导出、项目管理）
- 按类别筛选

## 六、数据同步机制

### 标注保存
- 保存标注时自动写入 YOLO 格式 `.txt` 文件（与图片同名）
- 保存前自动创建 `.backup` 备份文件
- 支持撤销/重做（最多100步）

### 数据集管理
- **添加数据集**: 注册到 `datasets.json` 配置文件，自动发现目录结构
- **重命名数据集**: 修改 `datasets.json` 中的显示名称
- **移除数据集**: 从 `datasets.json` 中删除注册信息，不影响磁盘文件
- **隐藏YAML数据集**: 添加到排除列表，可随时恢复
- **创建标注集**: 自动在数据集目录下创建新的标注目录
- **导出数据**: 生成 COCO/VOC/Mask/CSV 文件到 `exports/` 目录

### 文件结构
```
数据集目录/
├── images/
│   ├── train/          # 训练图片
│   └── val/            # 验证图片
├── labels/
│   ├── train/          # 训练标注 (.txt)
│   └── val/            # 验证标注 (.txt)
├── exports/            # 导出文件 (COCO/VOC/Mask/CSV)
└── data*.yaml          # 数据集配置
```

## 九、CellposeSAM 批量分割

### 单张预览
1. 设置直径 → 点击 **「⚡ 运行预览」**
2. 预览结果叠加显示在画布上
3. 满意则 **「✓ 全部保存到标注」**

### 批量分割
1. 设置起始/结束图片范围（或选择「全量」/「从当前到末尾」）
2. 选择是否「跳过已有标注」或「覆盖已有标注」
3. 点击 **「一键批量分割」**
4. 实时查看进度条和统计（保存/跳过/失败数量）
5. 完成后可选择「保存」或「不保存」

## 十、数据格式

### YOLO 标注格式
标注文件为 `.txt` 格式，与图片同名，位于标注目录下：
- **多边形**: `class_id x1 y1 x2 y2 x3 y3 ...`（归一化坐标 0-1）
- **矩形框**: `class_id cx cy w h`（归一化坐标 0-1）

### 数据集配置
系统自动扫描 `MEDSAM_ROOT` 目录下的 `data*.yaml` 文件：
```yaml
train: path/to/train/images
val: path/to/val/images
nc: 5
names:
  0: Neutrophil
  1: Lymphocyte
  2: Macrophage
  3: SEC
  4: RBC
```

## 十一、常见问题

| 问题 | 解决方法 |
|------|---------|
| 打开页面看不到更新 | 按 `Ctrl+Shift+F5` 强制刷新 |
| `ModuleNotFoundError: labeling_tool` | 必须从 `D:\VM_share` 目录启动 |
| AI功能报503错误 | 缺少模型文件或依赖项目 |
| 端口8000被占用 | 使用 `--port 8001` |
| GPU 内存不足 | 系统会自动回退到 CPU |
| 数据集列表为空 | 检查 `MEDSAM_ROOT` 下的 `data*.yaml` 或通过数据管理添加 |
| 标注保存失败 | 检查标注目录写入权限 |
| YOLO导出下载失败 | 检查浏览器是否阻止了文件下载 |

## 十二、AI 模型说明

| 模型 | 用途 | 首次加载 |
|------|------|---------|
| CellposeSAM | 自动细胞分割（无需标注） | ~15秒 |
| SAM3 | 基于提示的精细分割（框/文本/点） | ~30秒 |
| BiomedCLIP (Hybrid) | 自适应细胞分类 | ~10秒 |
| BiomedCLIP (Few-shot) | 少样本细胞分类 | ~10秒 |

所有模型首次使用时延迟加载，后续调用极快。GPU 内存不足时自动降级到 CPU。
